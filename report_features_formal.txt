Retrieval and Ranking
The system supports keyword-based retrieval over the integrated EV opinion corpus using Apache Solr as the backend retrieval engine. Query processing is configured with the `edismax` parser, which improves robustness for natural user queries, and ranking is influenced by field weighting in order to prioritise more meaningful matches. In particular, the `text` field is boosted relative to `source_target`, so documents with stronger content-level relevance are ranked more highly than those matching only metadata. This improves retrieval quality by ensuring that the system prioritises substantive opinion content rather than superficial field overlap.

Result Highlighting
To improve interpretability, the system highlights matched query terms directly within the returned result text. This allows users to quickly understand why a document was retrieved and where the relevance originates within the opinion itself. Highlighting is particularly useful in an opinion search setting, where result texts may be relatively long and users need to identify the precise part of the text that corresponds to their information need. The implementation also normalises returned values from Solr before rendering, ensuring that highlighting works consistently even when fields are returned as lists.

Enhanced Search and Result Interpretation
The application extends beyond a basic ranked list by providing additional views that help users interpret the overall result set. A sentiment-oriented insight panel summarises whether the current results are predominantly positive, negative, or neutral, and presents this information with colour cues for rapid interpretation. In addition, the visualization explorer allows users to switch between multiple analytical views of the current result set, including a thematic word cloud, platform distribution, and source/target distribution. These features support exploratory search by allowing users to move from individual retrieved documents to a broader understanding of trends, dominant themes, and result composition.

Interactive Refinement
The system includes lightweight interactive refinement through quick filters derived from the current search context. Rather than requiring users to reformulate the query repeatedly, the interface presents concise refinement options based on available platforms, source/target categories, and sentiment labels. This supports a more efficient exploratory workflow in which users can progressively narrow the result set while preserving the original search intent. The design is intentionally simple so that refinement behaves as an extension of search filtering rather than as a separate and potentially confusing interaction mode.

Multifaceted Search
Multifaceted search is a central feature of the application and enables users to refine results along several structured dimensions. The implemented facets include platform, source/target, dataset split, polarity, and subjectivity. This is important because EV opinion data is heterogeneous both in source and in tone. A user may wish to retrieve not only documents about a topic such as charging or safety, but specifically negative opinions from one platform, or opinionated rather than neutral content. By exposing these dimensions as filters, the system improves retrieval precision, reduces noise, and supports more targeted analysis of the corpus.

Pagination and Scalable Exploration
Pagination improves the usability of the system by dividing large result sets into manageable subsets. Instead of overwhelming users with all matches at once, the application presents a fixed number of results per page and allows navigation across the full set of retrieved documents. This preserves readability, improves interface responsiveness, and supports more systematic exploration of the corpus. Pagination also works together with ranking, filtering, and visualization features, allowing users to explore large result sets progressively without losing context.

Overall Contribution
Taken together, these features transform the application from a simple keyword retrieval interface into a more effective opinion retrieval system. Solr-based ranking improves relevance, highlighting improves transparency, enhanced search features improve interpretability, interactive refinement improves usability, multifaceted search improves precision, and pagination improves scalability. Collectively, these design choices make the system better suited for analysing public opinion on electric vehicles across multiple online platforms.
